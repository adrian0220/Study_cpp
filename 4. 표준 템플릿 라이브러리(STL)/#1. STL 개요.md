# 1. STL 개요

<br/>
<br/>

## 1) STL (Standard Template Library) 이란?

<br/>

C++에서 제공하는 표준 라이브러리 <br/>
프로그램에 필요한 자료구조와 알고리즘을 Template 형태로 제공한다. <br/>

<br/>

### STL의 특징

<br/>

- 미리 만들어진 클래스를 활용한 라이브러리 사용 <br/>
- STL 알고리즘은 독립적으로 사용된다. <br/>
- 전통적인 런타임 다형성에 비해 훨씬 효과적인 컴파일 타임 다형성을 제공 <br/>
- 라이브러리는 대부분 제너릭 프로그래밍에 기반하여 작성 됨 <br/>

<br/>
<br/>

## 2) STL의 구성요소

<br/>

### 2-1) 컨테이너 (Container)

<br/>

객체를 저장하는 객체 혹은 자료구조 <br/>
C++ 에선 자료구조로 구성되어있다. <br/> 

<br/>

__컨테이너의 종류__

<br/>

- 시퀀스 컨테이너 (Sequence Container) <br/>
- 연관 컨테이너 (Associative Conatainer) <br/>
- 컨테이너 어댑터 (Container Adapter) <br/>

<br/>

__1. 시퀀스 컨테이너 (Sequence Container)__

<br/>

데이터를 선형으로 저장하는 형식의 컨테이너 <br/>
특별한 제약이나 규칙이 없는 일반적인 형태의 컨테이너 <br/>
Queue, Vector, List, Dequeue 등이 여기에 포함 <br/>

<br/>

- 시퀀스 컨테이너의 요구 사항 <br/>

<br/>

STL에서 시퀀스 컨테이너는 기본 컨테이너의 개념에 다음과 같은 요구 사항을 추가하여 정의합니다. <br/>

<br/>

1. 모든 요소가 직선 순서대로 배치되어 있어야 한다. <br/>
첫 번째 요소와 마지막 요소를 제외한 나머지 요소들은 반드시 앞뒤로 인접한 요소를 하나씩 가지고 있어야 한다. <br/>

<br/> 

2. 반복자가 최소한 순방향 반복자(forward iterator) 이상이어야 한다. <br/>
이것은 반복자가 이동할 때마다 요소들의 순서가 변하지 않음을 보장해 준다. <br/>

<br/> 

3. 시퀀스 컨테이너의 요소들은 명확한 순서를 가지므로, 특정 위치를 참조하는 연산이 가능해야 한다. <br/>

<br/>

__2. 연관 컨테이너 (Associate Container)__

<br/>

키(key)와 값(value)처럼 관련있는 데이터를 하나의 쌍으로 저장하는 컨테이너 <br/>
키와 값을 이용한 연관 컨테이너를 통해 데이터의 빠른 접근이 가능 <br/>
하지만 연관 컨테이너는 삽입되는 요소의 위치를 지정할 수는 없음 <br/>
set, multiset, map, multimap 등이 여기에 포함 <br/>

<br/>

연관 컨테이너는 보통 균형 잡힌 이진 탐색 트리(balanced binary search tree)나 해시 테이블(hash table)을 이용하여 구현 <br/>

### 2-2) 반복자 (Iterator)

<br/>

STL 컨테이너에 저장된 요소를 반복적으로 순회하여 요소에 접근을 제공하는 객체 <br/>
컨테이너의 구조나 요소의 타입과 상관 없이 저장된 데이터를 순회하기 위한 제너릭 프로그래밍 <br/>

<br/>

__반복자의 요구사항__ <br/>

<br/>

- 반복자가 가리키는 요소의 값에 접근할 수 있어야 한다. 이때 참조 연산자를 사용한다. <br/>
- 반복자 사아의 대입, 비교 연산이 가능해야 한다. <br/>
- 가리키는 요소의 주변 요소로 이동이 가능해야 한다. <br/>
- 반복자의 위치를 판단할수 있도록 =, ==, != 와같은 반복자간의 대입, 비교 연산이 정의되어야 한다. <br/>

<br/>
 
__반복자의 종류__

<br/>

1. 입력 반복자(input iterator)
2. 출력 반복자(output iterator)
3. 순방향 반복자(forward iterator)
4. 양방향 반복자(bidirectional iterator)
5. 임의 접근 반복자(random access iterator)

<br/>

__1. 입력 반복자 (Input Iterator)__ <br/>

<br/>

가장 단순한 형태의 반복자 <br/>
컨테이너로부터 값을 읽는 데 사용 <br/>
프로그램이 그 값을 변경 할 수 없으므로 읽기 전용 알고리즘에 사용 <br/>

<br/>

1. 증가 연산자를 사용하여 순방향으로만 이동 <br/>
2. 참조 연산자를 사용하여 반복해서 요소 참조 가능 <br/>
 
<br/>

__2. 출력 반복자 (Output Iterator)__ <br/>

<br/>

컨테이너의 값을 변경하는 데 사용 <br/>
프로그램에서 값을 변경할 수 있으나 읽을 수는 없다. <br/>
쓰기 전용 알고리즘에서 사용 <br/>

<br/>

1. 증가 연산자를 사용하여 순방향으로만 이동<br/>
2. 참조 연산자를 사용하여 단 한 번만 요소에 값을 입력 <br/>

<br/>

__3. 순방향 반복자 (Foward Iterator)__ <br/>

<br/>

입출력이 모두 가능한 반복자 <br/>
다중 패스 알고리즘에 사용 가능 <br/>

<br/>

1. 증가 연산자를 사용하여 순방향으로만 이동 <br/>
2. 참조 연산자를 사용하여 여러 번 반복해서 같은 요소에 참조, 변경이 가능 <br/>
3. 입력 반복자와 출력 반복자의 기능 모두 포함 <br/>

<br/>

__4. 양방향 반복자 (Bidirectional Iterator)__ <br/>

<br/>

입출력이 모두 가능한 반복자 <br/>

<br/>

1. 증가 연산자(++)를 사용하면 순방향으로, 감소 연산자(--)를 사용하면 역방향으로도 이동 <br/>
2. 참조 연산자를 사용하여 여러 번 반복해서 같은 요소 참조, 변경이 가능 <br/>
3. 양방향 반복자는 순방향 반복자의 기능을 모두 포함한다. <br/>

<br/>

__5. 임의 접근 반복자 (Random Access Iterator)__ <br/>

<br/>

최상위 레벨의 반복자로서 많은 기능을 제공하는 반복자 <br/>

<br/>

1. 양방향 반복자의 모든 기능을 포함하고 있고, 첨자 연산자([])를 통해 임의의 요소에 접근 가능 <br/>
2. 증감 연산자를 통해 양방향으로 이동할 수 있고, 요소의 순서를 결정하기 위해 관계 연산자를 사용 <br/>
3. 임의 접근 반복자는 양방향 반복자의 모든 기능과 함께 다른 반복자의 일 모두 가능 <br/>

<br/>
<br/>

__반복자 계층__ <br/>

<br/>

STL에서 반복자는 계층적으로 분류된다. <br/>
예를 들어, 순방향 반복자는 입력 반복자와 출력 반복자의 모든 기능을 포함한다. <br/>

__기타 반복자__ <br/>

<br/>

1. 스트림 반복자 (Stream Iterator) <br/>
반복자 연산을 통해 알고리즘이 입출력 스트림에 쉽게 접근할 수 있도록 해준다. <br/>

<br/>

2. 삽입 반복자 (Insert Iterator) <br/>

### 2-3) 알고리즘 (Algorithm)

<br/>

정렬, 삭제, 검색, 연산 등을 해결하는 일반화된 방법을 제공하는 함수 템플릿 <br/>

<br/>

__STL 알고리즘의 특징__ <br/>

<br/>

- C++ 에서 STL을 제공하는 지 목적은 알고리즘에 대한 효율적인 구현이다. <br/>
- STL 알고리즘 함수나 STL 컨테이너의 멤버 함수를 사용하여 구현이 가능하다. <br/>
- STL 컨테이너는 알고리즘을 제공하는 많은 전역 함수와 함께 사용해야만 제 기능을 발휘한다.<br/>
- STL 알고리즘 함수는 반복자를 통해 임의의 컨테이너에 같은 방법으로 적용된다. <br/>
- 대부분의 알고리즘 함수는 algorithm 헤더 파일과 numeric 헤더 파일에 정의되어 있다. <br/>

<br/>

__1. 읽기 알고리즘__ <br/>

<br/>

- 컨테이너를 변경하지 않고, 컨테이너의 지정된 범위에서 특정 데이터를 읽기만 하는 함수 <br/>
- find() : 두 개의 입력 반복자로 지정된 범위에서 특정 값을 가지는 첫 번째 요소를 가리키는 입력 반복자 반환 <br/>
- for_each() : 두 개의 입력 반복자로 지정된 범위의 모든 요소를 함수 객체에 대입한 뒤 객에 반환 <br/>

<br/>

__2. 변경 알고리즘__ <br/>

<br/>

- 컨테이너의 지정된 범위에서 요소 값을 변경 하는 함수 <br/>
- copy() : 두 개의 입력 반복자로 지정된 범위의 모든 요소를 출력 반복자가 가리키는 위치에 복사 <br/>
- swap() : 두 개의 참조가 가리키는 위치의 값을 서로 교환 <br/>
- transform() : 두 개의 입력 반복자로 지정된 범위의 모든 요소를 함수 객체에 대입한 뒤, 출력 반복자가 가리키는 위치에 복사 <br/>

<br/>

__3. 정렬 알고리즘__ <br/>

<br/>

- 컨테이너의 지정된 범위의 요소들이 정렬되도록 컨테이너 변경 <br/>
- 올바른 정렬을 위해 입의 접근 반복자 사용 <br/>
- sort() : 두 개의 임의 접근 반복자로 지정된 범위의 모든 요소를 서로 비교하여, 오름차순으로 정렬 <br/>
- stable_sort() : 두 개의 임의 접근 반복자로 지정된 범위의 모든 요소를 서로 비교하여, 값이 서로 같은 요소들의 상대적인 순서는 유지하면서 오름차순으로 정렬 <br/>
- binary_search() : 오름차순으로 정렬한 후에, 전달된 값과 같은 값이 있으면 참(true)을 반환하고 없으면 거짓(false)을 반환<br/>

<br/>

__4. 수치 알고리즘__ <br/>

<br/>

- 수치 알고리즘 함수는 다른 알고리즘 함수와는 달리 numeric 헤더 파일에 정의 <br/>
- accumulate() : 두 개의 입력 반복자로 지정된 범위의 모든 요소의 합을 반환 <br/>

<br/>
<br/>

### 2-4) 함수 객체 (Function Class)

<br/>

함수처럼 동작하는 객체로 operator() 연산자를 통해 오버로딩 한 객체 <br/>
STL 알고리즘이 데이터를 처리하기 위한 매개변수 <br/>

<br/>

__함수 객체의 장점__

<br/>

- 함수 객체는 상태(state)를 포함할 수 있습니다. <br/>
- 함수 객체는 타입이므로, 템플릿 인수로 사용할 수 있습니다 <br/>

<br/>
